Building Probability of Default Model on Fannie Mae acquired loans

1) Description:

Building probabiliyt of default model using Fannie Mae Single Family Fixed Rate Mortgage Datasets. Fannie Mae provides loan acquisition and performance data on a portion of its single-family mortgage loans to promote better understanding of the credit performance of Fannie Mae mortgage loans. The population includes two datasets.

The datasets consist of two tables - (i) Acquisition Data & (ii) Performance Data. Every quarter, the repository is updated to add another quarter of acquisitions and performances.

Reference Link: - Fannie Mae

2) Variable Definition

Definition of few key metrics used in model development

  • Loan_IDENTIFIER: Unique key for borrowers
  • Default Flag: Whether a loan defaults in 12 months after acquired by Fannie Mae. 0 if the loan is performing, 1 if the laon defaults.
  • Loan to Value(LTV) - Reflects the loan-to-value ratio of the loan amount secured by a mortgaged property on the origination date of the underlying mortgage loa
  • Combined Loan to Value(LTV) - Reflects the loan-to-value ratio inclusive of all loans secured by a mortgaged property on the origination date of the underlying mortgage loan.
  • Debt to Income Ratio - Ratio of borrower’s total monthly obligations (including housing expense) by his or her stable monthly income
  • Loan Purpose: Indicator denoting type of mortgage loan
    • P - Purchase Mortgage
    • C - Cash Out Mortgage
    • R - No Cash Out Mortgage
    • U - Refinance Mortgage Purpose may be the purchase of a new property or refinance of an existing lien (with cash out or with no cash out).

3) Event Definition - Default

Borrowers are said to be non perfrofming or defaulting the loan when they have missed the monthly paymnets for 90 days(3 Months). But since majority of Fannie Mae acquired mortgage are prime loans, the number of defaults using conventional definition is too low(< 0.05%).

Hence, for this case study, the default is taken as loan that failed to meet the monthly payment obligations for two months or more.

4) Key Challenges

Main obstacles while developing the default model were:

  • Imbalanced Dataset - Fannie Mae has specific criteria basis which a loan is acquired. The acquired loans are then bundled together and sold as Securities(MBS) to general public. Hence, majority of the loans are of prime lending resulting in very low number of defaulters.
  • Absence of key informations - Since Fannie Mae is providing this data publically, the information captured are very superficial thus impacting the performance of model developed.

def getclasses(df):
    i = 0
    F = pd.DataFrame()
    for c in df.select_dtypes(include = ['object']).columns:
        if c != 'mortgage_insurance_type':
            t_i = pd.DataFrame(df[c].unique()).rename(columns={0: c}).sort_values(by=c).reset_index()
        else:
            t_i = pd.DataFrame(df[c].unique()).rename(columns={0: c}).reset_index()            
        
        F = pd.concat([F, t_i.iloc[:,1]], axis=1)
        i += 1
            
    F = F.replace(np.nan, '', regex=True).T

    NClass = []
    for r in range(0,len(F)):
        i = 0
        robs = F.iloc[r,:]
        for c in robs:
            if (c == ""):
                continue
            else:
                i +=1
        NClass.append(i)

    df2 = F
    df2['NClass'] = NClass

    cols = df2.columns.tolist()
    cols = cols[-1:] + cols[:-1]
    df2 = df2[cols]
    return df2.sort_values('NClass', ascending = False)

def missing_values_table(df):
        mis_val = df.isnull().sum()
        mis_val_percent = 100 * df.isnull().sum() / len(df)
        mis_val_table = pd.concat([mis_val, mis_val_percent], axis=1)
        mis_val_table_ren_columns = mis_val_table.rename(
        columns = {0 : 'Missing Values', 1 : '% of Total Values'})
        
        mis_val_table_ren_columns = mis_val_table_ren_columns[
            mis_val_table_ren_columns.iloc[:,0] != 0].sort_values('% of Total Values', ascending=False).round(1)
        
        print ("Given data has " + str(df.shape[1]) + " columns.\n" +
            str(mis_val_table_ren_columns.shape[0]) + "columns have missing values with following frequencies.")
        
        # Return the dataframe with missing information
        return mis_val_table_ren_columns

8) Feature Engineering

Following changes were made to the dataset to have a more representative sample.

  • Creating New Fields Basis Domain Knowledge - Formulated aggregated credit score as FICO instead of having two fields - Borrower's and Coborrower's Credit Score. The formulated varaible had a correlation of 95.87% with the Borrower's Credit Score ensuring that the variance/insight is not lost.
  • Removed Fields that doesnt affect the loan approval UW process - Basis US Fair Lending Act(Truth in Lending Act), ECOA prohibits discrimination based on race, color, religion, national origin, demographs, sex, marital status, age, source of income or whether a person exercises rights granted under the Consumer Credit Protection Act for any credit transaction and through the life of the loan.
  • Polynomial Features - Non-Lilnear features were created basis the most relevant fields in the data. However, the new features were not contributing to enhancement of model performance and hence removed.

Model Development - Logistic Regression - Synthetic Minority Over Sampling Technique(SMOTE)

This dataset is a typical example of an imbalanced classification. This is a scenario where the number of observations belonging to one class is significantly lower than those belonging to the other classes.In this situation, the predictive model developed using conventional machine learning algorithms could be biased and inaccurate.

This happens because Machine Learning Algorithms are usually designed to improve accuracy by reducing the error. Thus, they do not take into account the class distribution / proportion or balance of classe and predicts all the samples to be from majority calss.

SMOTE is a technique where data is taken from the minority class as an example and then new synthetic similar instances are created. The created samples are are created basis the linearity of the given minority class. These synthetic instances are then added to the original dataset. The new dataset is used as a sample to train the classification models.

Using Cross Validation to get Best Fit

Steps invloved in K-Fold Cross Validation -

  • Shuffle the dataset randomly.
  • Split the dataset into k groups

  • For each unique group:
    • Take the group as a hold out or test data set
    • Take the remaining groups as a training data set
    • Fit a model on the training set and evaluate it on the test set
    • Retain the evaluation score and discard the model
    • Summarize the skill of the model using the sample of model evaluation scores
